# SMAGo — Самомодифицирующийся ИИ-агент

Легковесный Go-агент, общающийся через Telegram, работающий с любым OpenAI-совместимым LLM, хранящий историю диалогов в SQLite и способный модифицировать собственный исходный код и бинарник прямо во время работы.

SMAGo работает как приложение в области системных значков Windows с супервизором, который автоматически перезапускает агента при сбое и управляет обновлениями методом горячей замены.

> **Написан целиком ИИ.** Minimax M3 и DeepSeek через OpenCode создали архитектуру и бойлерплейт. Далее SMAGo через MiMo V2.5 (vision) развивался самостоятельно. Роль человека: идея, направление, ревью кода.

> 🇬🇧 *Read in English: [README.md](README.md)*

## Возможности

- **Telegram-бот** — long-polling, без вебхуков
- **Мультипровайдерный LLM** — OpenCode, DeepSeek, llama.cpp или любое OpenAI-совместимое API
- **Вызов инструментов** — терминал, чтение/запись/редактирование файлов, веб-поиск, распознавание изображений
- **Самомодификация** — обновление, откат, перезапуск через инструмент `self_modify` или команды Telegram
- **Супервизор** — значок в области системных значков, автоперезапуск при аварии, замена версий с защитой от «плохих» сборок
- **Рендеринг Markdown** — заголовки жирным, таблицы с выравниванием, блоки кода
- **Индикатор набора** — бот показывает «печатает...» во время обработки
- **Управление сессиями** — хранение в SQLite, несколько сессий, привязка к чатам
- **DCP** — динамическое сокращение контекста для укладывания в окно модели
- **Стоп/прерывание** — корректное или принудительное прерывание длительных задач

## Настройка и запуск

### Требования

- **Windows 10/11**
- **Go 1.26+** ([скачать](https://go.dev/dl/))
- **Git**
- **Токен Telegram-бота** — получить у [@BotFather](https://t.me/BotFather)

### Быстрая установка (winget)

```cmd
winget install --id GoLang.Go -e --source winget
winget install --id Git.Git -e --source winget
```

Или скачайте вручную с [go.dev](https://go.dev/dl/) и [git-scm.com](https://git-scm.com/).


### 1. Клонирование репозитория

```cmd
git clone git@github.com:AsmanovLev/SMAGo.git
cd SMAGo
```

### 2. Сборка

```cmd
build.bat
```

Создаёт три бинарника в `bin/`:

| Бинарник | Описание |
|----------|----------|
| `bin/agent.exe` | Консольная сборка (для отладки) |
| `bin/smago-bg.exe` | Фоновая сборка (без окна консоли) |
| `bin/supervisor-bg.exe` | Супервизор со значком в области системных значков |

### 3. Конфигурация

```cmd
copy config.example.json config.json
```

Откройте `config.json` и заполните:

| Поле | Описание |
|------|----------|
| `telegramToken` | Токен бота от @BotFather |
| `telegramChatID` | Ваш Telegram chat ID (см. ниже) |
| `provider` | Имя провайдера LLM (по умолчанию: `opencode-go`) |
| `defaultModel` | Имя модели (по умолчанию: `mimo-v2.5`) |
| `providers.*.apiKey` | API-ключ выбранного провайдера |

#### Как узнать свой Telegram Chat ID

1. Начните диалог с ботом в Telegram
2. Отправьте любое сообщение
3. Откройте `data/smago.log` и найдите chat ID, или воспользуйтесь [@userinfobot](https://t.me/userinfobot)

#### Переменные окружения (альтернатива, для секретов)

Храните конфиденциальные значения отдельно от `config.json`:

```cmd
set SMAGO_TELEGRAM_TOKEN=ваш_токен
set SMAGO_TELEGRAM_CHAT_ID=ваш_chat_id
set SMAGO_OPENCODE_KEY=ваш_api_ключ
```

### 4. Запуск

**С супервизором (рекомендуется):**

```cmd
start-supervised.bat
```

Супервизор работает молча в области системных значков. ПКМ по значку — параметры.

**Без супервизора:**

```cmd
bin\smago-bg.exe
```

**Отладка (с выводом в консоль):**

```cmd
bin\agent.exe
```

### 5. Проверка

Отправьте `/start` боту в Telegram. Должно появиться справочное сообщение.

## MCP (Model Context Protocol)

SMAGo поддерживает внешние серверы инструментов через [MCP](https://modelcontextprotocol.io/).
Любой stdio-based MCP-сервер работает — Node.js, Python, Go, Rust.

### Как это работает

1. Вы добавляете MCP-сервер в `config.json`
2. SMAGo запускает процесс сервера при старте
3. Инструменты сервера появляются как `<сервер>__<инструмент>` (например `fs__read_file`)
4. LLM может вызывать их так же, как встроенные инструменты

### Настройка

1. Положите файлы MCP-сервера в `mcps/` (игнорируется git):

```cmd
mkdir mcps\my-server
```

2. Добавьте в `config.json`:

```json
{
  "mcp": {
    "fs": {
      "command": ["npx", "-y", "@modelcontextprotocol/server-filesystem", "C:\\путь\\к\\папке"],
      "enabled": true
    }
  }
}
```

3. Перезапустите SMAGo. Новые инструменты появятся в `/tools`.

### Поля конфигурации

| Поле | Описание |
|------|----------|
| `command` | Массив: исполняемый файл + аргументы (через stdio) |
| `enabled` | `true` — подключаться, `false` — пропустить |
| `env` | Необязательные переменные окружения для процесса |

### Примечания

- Максимум 10 инструментов на сервер (самые важные, в порядке объявления)
- Логи сервера попадают в `data/smago.log`
- Отключённые серверы полностью пропускаются
- Больше примеров в `mcps/README.md`

## Управление версиями

SMAGo использует git commit SHA как идентификаторы версий:

```
data/
  versions/
    cff3262/agent.exe
    addc8d7/agent.exe
  current.json    → {"version": "cff3262"}
  next.json       → {"version": "addc8d7"} (ожидает замены)
```

Супервизор отслеживает появление `next.json` и корректно заменяет бинарник. Если новая версия падает в течение 20 секунд, она помечается как «плохая» и больше не используется.

### Boilerplate (`2850fcb`)

Рабочий прототип, написанный за одну сессию Minimax M3 и DeepSeek:

- **Telegram-бот** через long-polling (стандартный `net/http`, ноль внешних зависимостей)
- **LLM** — OpenAI-совместимые chat completions (работал с локальным llama.cpp, self-hosted эндпоинтами, OpenCode)
- **4 инструмента** — `bash` (shell), `read_file`, `write_file`, `list_dir`
- **Vision** — анализ изображений через мультимодальную модель (mimo-v2.5)
- **SQLite-сессии** — история диалогов в привязке к чату (modernc.org/sqlite, без CGO)
- **Markdown → HTML** — заголовки, жирный, курсив, блоки кода для Telegram
- **Самомодификация** — агент мог редактировать собственный Go-код, перекомпилировать и заменять бинарник
- **Супервизор** — приложение в области системных значков с автоперезапуском при аварии
- **Один бинарник** — без Docker, без WSL, без внешних сервисов

Провайдеры были перенесены из `opencode.json` автора: локальный llama.cpp, домашний сервер и self-hosted DeepSeek-прокси.

### Самостоятельная разработка (`34ede0a` — `0001437`)

**Что SMAGo построил самостоятельно:**


- **Git-интеграция и самообновление** (`34ede0a`) — `git.go`: чтение git-истории, показ диффов, использование commit SHA как идентификаторов версий
- **Прерывание и трассировка** (`76cb51d` — `b8d628c`) — команды `/stop` и `/abort`, компактный формат трассировки
- **Мажорный рефакторинг** (`83e9e0a`) — переход с последовательных номеров версий на **git commit SHA**, древовидный формат трассировки инструментов с аннотациями, безмолвные уведомления, команда супервизора `/rebuild`
- **Управление мультисессиями** (`9d94ad8`) — несколько именованных сессий на один чат, аннотации вызовов инструментов, подтверждение самообновления
- **DCP — Динамическое сокращение контекста** (`3d081b5`) — самая большая фича: `/compress`, стратегии сокращения (дедупликация, очистка ошибок, нUDGE системного промпта), автоматический расчёт лимитов на основе контекстного окна модели, повтор при HTTP 503/502/429/500 с экспоненциальной задержкой
- **Доведение сессий до ума** (`b8bc85a` — `c6eff72`) — `/rename` с автогенерацией имён через LLM, `/sessions`, `/switch`, `/delete`, белый список команд во время выполнения задачи, насыщенная `/help`
- **Реестр команд** (`2dc9382`) — `/help` и меню бота генерируются автоматически из определений команд

**Что OpenCode чинил когда SMAGo ломался:**

- Три сессии восстановления, когда самодеятельные модификации агента приводили к ошибкам сборки или авариям в рантайме — каждый раз OpenCode восстанавливал кодовую базу до рабочего состояния
